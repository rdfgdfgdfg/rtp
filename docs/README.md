# 简介

rtp 递归线程池，具有递归结构的线程池  
建议用它来运行有递归结构的多线程函数  
**当每个线程的开销大，无限递归，达成一定条件后结束：使用此线程池性能收益大**  
**当必须使用多线程，且难以转换为非多线程算法时：使用此线程池节省开发时间**  
------
| 概念 | 解释
|---|---
|线程树 | rtp是树结构，线程树指rtp的结构部分。线程树是一般的树
|线程节点 | 线程树中的节点，也是逻辑上在运行的线程。彻底运行完毕后会被移除（不析构）
| 节点容器 | 每一个线程节点都会有一个容器用于存放它的子节点，该容器名为节点容器
| 用户线程类 | 用户派生线程节点的类
| 用户函数 | 用户定义的需要执行的函数，类型为 `void (MAT::TTNode::\*)()`即`MAT::Fptr`，在用户线程类中声明
| 系统线程 | rtp中实际在运行的线程，使用`std::thread`实现
